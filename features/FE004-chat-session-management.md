# 会话管理 (Chat Session Management)

## 特性定义
会话管理负责管理用户与 AI 之间的对话历史。它确保对话状态在刷新页面后依然可见，并允许用户切换不同的会话。

## 业务逻辑
1. **自动初始化**：进入聊天页时，系统自动拉取最近一次活跃的会话。如果没有任何历史记录，则生成一个新的 UUID 作为 `sessionId`。
2. **消息流式处理**：前端通过 `HttpClient` 发起 `POST` 请求，并使用 `observe: 'events'` 捕获流式数据（SSE）。
3. **列表自动刷新**：当新会话产生第一条回复（`aiMsgIndex === 1`）并响应结束时，自动刷新侧边栏列表。

## 涉及子工程
- **ms-ng-view**: 前端 UI 与流式数据处理。
- **ms-py-agent**: 智能体逻辑与历史记录入库。
- **ms-java-biz**: 历史记录与会话列表的查询接口。

## 关键修复记录

### 2026-04-27：修复新会话产生重复侧边栏条目或多次刷新的问题
- **现象**：当新会话发起第一次请求时，侧边栏会多次刷新。
- **根因**：在流式 HTTP 请求中，`DownloadProgress` 事件会触发多次。原逻辑将 `refreshSessionsListSilently()` 放在了事件类型判断之外，导致每一个数据块到达时都会触发刷新。
- **解决方案**：将刷新逻辑移入 `HttpEventType.Response` 分支，确保仅在整个请求彻底完成后执行一次刷新。
